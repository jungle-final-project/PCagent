# 관리자 공통 진입점 안정화 체크리스트

이 문서는 5번 담당자의 첫 PR 범위를 작게 유지하기 위한 작업 체크리스트입니다. 목표는 관리자 화면의 권한 확인 위치를 명확히 만들고, AdminShell은 layout/navigation 책임만 유지하는 것입니다.

## 1. 작업 범위 확인

- [ ] 이번 PR 목표를 "관리자 공통 진입점 안정화"로 제한한다.
- [ ] 5번 담당 영역인지 확인한다.
- [ ] API 요청/응답 구조를 바꾸지 않는 작업인지 확인한다.
- [ ] OpenAPI 수정이 필요 없는 작업인지 확인한다.
- [ ] CI, Docker Compose, health smoke는 변경하지 않는다고 정한다.

## 2. 수정 파일 확정

- [ ] `apps/web/src/App.tsx`에서 관리자 route guard 적용 위치를 확인한다.
- [ ] `apps/web/src/features/auth/RequireAdmin.tsx`에서 관리자 권한 확인 skeleton을 둔다.
- [ ] `apps/web/src/components/layout/AdminShell.tsx`는 layout/navigation 책임만 유지한다.
- [ ] `apps/web/src/features/admin/pages/AdminDashboardPage.tsx`는 5번 소유 dashboard이므로 필요한 경우에만 최소 수정한다.
- [ ] `.github/workflows/ci.yml`은 이번 PR에서 수정하지 않는다.
- [ ] `compose.yaml`은 이번 PR에서 수정하지 않는다.
- [ ] `tools/validate_openapi.py`는 이번 PR에서 수정하지 않는다.

## 3. 관리자 guard 구현

- [ ] `RequireAdmin` 컴포넌트를 auth feature 안에 둔다.
- [ ] `children: ReactNode`를 받아 권한이 있으면 그대로 렌더링한다.
- [ ] Sprint 1에서는 `localStorage.getItem('buildgraph.token')` 기준으로만 판정한다.
- [ ] demo admin token 문자열은 `demo-jwt-admin` 기준으로 확인한다.
- [ ] 권한이 없으면 redirect하지 않고 안내 화면을 보여준다.
- [ ] 안내 화면은 `Screen`을 사용해 기존 사용자 화면 shell과 맞춘다.
- [ ] 안내 화면에는 "관리자 권한이 필요합니다"라는 명확한 제목을 둔다.
- [ ] 안내 화면에는 로그인 이동 버튼을 둔다.
- [ ] 안내 화면에는 홈 이동 버튼을 둔다.
- [ ] 안내 화면에는 내부 구현 설명보다 사용자 행동 중심 문구를 사용한다.

## 4. Route 적용

- [ ] `App.tsx`에서 `RequireAdmin`을 import한다.
- [ ] `/admin` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/agent-sessions/:id` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/tool-invocations/:id` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/rag-evidence/:id` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/parts` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/as-tickets` route element를 `RequireAdmin`으로 감싼다.
- [ ] `/admin/as-tickets/:ticketId` route element를 `RequireAdmin`으로 감싼다.
- [ ] 일반 사용자 route는 건드리지 않는다.
- [ ] fallback route는 건드리지 않는다.

## 5. AdminShell 책임 확인

- [ ] AdminShell에 auth/permission 판단 코드를 넣지 않는다.
- [ ] AdminShell은 sidebar, header, main layout만 담당하게 둔다.
- [ ] AdminSidebar navigation 항목은 그대로 유지한다.
- [ ] 도메인별 관리자 page 데이터를 AdminShell로 끌어올리지 않는다.
- [ ] shell 공통 버튼은 이번 PR에서 동작 구현하지 않는다.

- [ ] localStorage에 `buildgraph.token=demo-jwt-admin`을 넣고 `/admin`에 접근한다.
- [ ] 관리자 dashboard가 보이는지 확인한다.
- [ ] `/admin/parts` 접근 시 guard가 먼저 적용되는지 확인한다.
- [ ] `/admin/as-tickets` 접근 시 guard가 먼저 적용되는지 확인한다.
